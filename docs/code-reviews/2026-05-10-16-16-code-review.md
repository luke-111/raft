## 🟡Important 问题

### 1. 模块 Go 版本与工具链兼容性

- 严重级别：Important
- 问题类别：工程规范
- 位置：`src/github.com/cmu440/go.mod:3`
- 问题描述：`go 1.25.0` 要求较新的 Go 工具链；若教学环境、CI 或同学本地仍为 1.22–1.24，可能出现无法下载模块或编译失败。
- 影响：拉代码即构建失败或评分环境不一致，阻塞协作与提交。
- 建议：与课程官方要求的 Go 版本对齐；若 1.25 为刻意选择，在 README 中写明最低版本与安装方式。
- 证据：`go.mod` 仅三行，当前字段为 `go 1.25.0`。

### 2. 日志冲突时的回退策略偏保守

- 严重级别：Important
- 问题类别：性能
- 位置：`src/github.com/cmu440/raft/raft.go` 中 `replicatePeer` 失败分支（约 438-451 行）
- 问题描述：`AppendEntries` 被拒绝时每次将 `nextIndex[server]` 减 1；未实现论文常见的冲突项任期回退或批量跳过，RPC 失败时本轮 `replicatePeer` 直接退出。
- 影响：在日志很长或频繁分区时，复制与提交延迟可能显著上升；极端情况下依赖心跳周期多次重试，测试可能仍过但生产类比场景体验差。
- 建议：后续迭代引入按任期/索引的快速回退或类似 `TestBackup` 类作业要求中的优化；RPC 失败可考虑与心跳解耦的重试策略（需注意与 leader 任期检查一致）。
- 证据：`reply.Success` 为 false 时 `nextIndex[server]--`；`sendAppendEntries` 为 false 时 `return`。

## 🔵Minor 问题

### 1. 使用已弃用的 `ioutil`

- 严重级别：Minor
- 问题类别：工程规范
- 位置：`src/github.com/cmu440/raft/raft.go:33`、`raft.go:144`
- 问题描述：`import "io/ioutil"` 与 `ioutil.Discard` 在较新 Go 中已弃用。
- 影响：未来 Go 版本移除该包时需改动；静态分析告警。
- 建议：改为 `io.Discard` 并删除 `ioutil` 导入。
- 证据：源码 import 与 `log.New(ioutil.Discard, "", 0)`。

### 2. RPC 客户端错误日志文案不准确

- 严重级别：Minor
- 问题类别：可维护性
- 位置：`src/github.com/cmu440/rpc/rpc.go`（`ClientEnd.Call` 编码失败分支，约 97-98 行）
- 问题描述：日志字符串写为 `encode reply`，实际为编码请求参数。
- 影响：排障时误导方向。
- 建议：改为 `encode args` 或等价描述。
- 证据：`qe.Encode(args)` 与 `log.Fatalf("ClientEnd.Call(): encode reply: ...` 不一致。

### 3. 测试中无意义的空临界区

- 严重级别：Minor
- 问题类别：工程规范
- 位置：`src/github.com/cmu440/raft/raft_test.go` 约 387-389 行
- 问题描述：`cfg.mu.Lock()` 后立即 `cfg.mu.Unlock()`，中间无共享状态访问。
- 影响：增加阅读成本，易被误认为遗漏逻辑。
- 建议：删除该对调用或补充注释说明意图。
- 证据：`start1` 内连续 `Lock`/`Unlock`。

### 4. 调试日志文件权限偏宽

- 严重级别：Minor
- 问题类别：安全性 / 工程规范
- 位置：`src/github.com/cmu440/raft/raft.go` 中 `os.OpenFile`（约 136 行）
- 问题描述：日志文件使用 `0755` 权限位创建。
- 影响：多用户主机上其他本地用户可能可读日志（课程单机影响小）。
- 建议：日志文件使用 `0600` 或 `0640`。
- 证据：`os.O_CREATE` 与 `0755` 常量在 `OpenFile` 调用中。
